POA case study

Online Courses Platform (POA)

Full-stack SaaS platform currently developed and operated end-to-end, with video delivery, administrative workflows, subscription management, AI-assisted analytics, and runtime configuration controls.

Problem
Deliver and run an online education SaaS with robust payments, secure content, and scalable operations.
Role
Current solo execution: conception, architecture, frontend, backend, infra, deploy, and production support.
Stack
TypeScript, React 18, Node/Express, PostgreSQL, Drizzle, Claude/Codex/LLM Gateway, Object Storage, Webhooks.

Technical Focus

  • Shared schema and end-to-end typing between client and server with runtime validation.
  • Idempotent payment webhooks with deduplication and manual reprocessing through admin.
  • Internal AI gateway for per-feature model routing, cost auditing, and safe extensibility.
  • Context-aware chatbots with dataset parsing (CSV, XLSX, SPSS) for data-grounded responses.
  • Server-side PDF generation with user watermarking for content protection.
  • Feature flags and dynamic configuration for runtime rollout without redeploy.
  • Observability pipeline for frontend and backend error capture, aggregation, and alerts.
