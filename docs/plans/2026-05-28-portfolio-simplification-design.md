# Portfolio Simplification Design

## Goal

Redesign the portfolio around a strong photo-based hero while making the rest of the page simpler, more editorial, and less template-like.

## Direction

Keep the current dark hero with Thiago's photo as the main identity element. Reduce visual noise after the hero so the site feels like a focused engineering portfolio instead of a generated landing page.

## Approved Structure

1. Hero with photo
2. Stack / specialties
3. Selected projects
4. Short experience summary
5. Certification
6. Simple contact

## Hero

The hero stays visually prominent. It should keep the photo, dark overlay, name, location, and a primary contact CTA.

Adjustments:
- Use a fixed role statement instead of typing animation on mobile.
- Keep one strong primary button.
- Convert the resume action into a quieter secondary link or lighter button.
- Reduce particles and glow so the photo remains the focal point.

## Navigation

Simplify navigation to the sections that matter:
- Home
- Stack
- Projects
- Experience
- Contact

Remove Blog from primary navigation for now. On mobile, remove the seven-item bottom navigation or replace it with a much smaller four-item version.

## Content Density

The current page is too long, especially on mobile. Reduce the page by cutting duplicate and low-signal surfaces.

Remove or hide:
- Project filters
- Project carousels on cards
- Project modal/detail page for now
- Simulated contact form
- Floating scroll-to-top button
- Long timeline presentation
- Blog from the main journey

## Projects

Show three selected projects maximum on the home page. Each project should be a simple card with:
- Title
- One short description
- Stack tags
- One optional link

Do not use carousels in project cards. If case studies are added later, they should be separate and intentional.

## Experience

Use a compact experience section instead of a long timeline. Show only the most relevant experiences on the home page, with two bullets each.

The full resume can live in a PDF or external link.

## Contact

Replace the large form with a compact contact block:
- Email
- LinkedIn
- GitHub
- Optional WhatsApp or calendar link

Only add a form when it sends to a real backend or service.

## Visual Style

Keep:
- Dark theme
- Hero photo
- Inter typography
- Red as the main accent

Reduce:
- Yellow accent usage
- Neon glow
- Icon volume
- Heavy card styling
- Motion and auto-animations

## Success Criteria

- The hero remains visually memorable.
- The rest of the page feels quieter and more professional.
- Mobile page length is materially shorter.
- Main navigation has fewer choices.
- Projects and contact are reachable faster.
- The page no longer feels like a template or AI-generated landing page.
